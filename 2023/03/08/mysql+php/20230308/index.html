2.pymysql.err.OperationalError: (1044

pymysql.err.OperationalError: (1044, “Access denied for user ‘testuser‘@’%’ to database ‘testdb’”)

这个是因为用户权限，创建用户后没有给予权限（指增删查改之类的）就会报这个错

关于权限的给予这里不再描述了（因为内容有点多），网上有很多文章

4.define(): Argument #3 ($case_insensitive)

警告很直接，因为 define定义常量在php8中已经无法不区分大小写了，即不再拥有第三个参数了

5.ERROR 1064 (42000): You have an error in your SQL syntax

ERROR 1064 (42000): You have an error in your SQL syntax; check the manual that corresponds to your MySQL server version for the right syntax to use near ‘{

我这里是 create table的时候出现的，检查后发现是 create应该用 ()而不是 {},在网上搜的时候发现有因为

在MySQL中，为了区分MySQL的关键字与普通字符，MySQL引入了一个反引号，而在定义名称时，应该用 ``,而不是 ''

即 'package_code' varchar(6) NOTNULL变为 `package_code` varchar(6) NOTNULL

但个人认为其实触发这个报错的大部分原因是出现了中文字符，改完就好了

7.ERROR 1366 (HY000): Incorrect string value

网上搜了很多，都是说数据库编码问题的，即要把数据库设置为utf-8，但是我的数据库就是这个编码，所以错误很明显不是这个

找了很久，找到了这篇文章讲的很好

总结一下：在你用cmd进行mysql操作的时候，可以查看其属性，会发现默认是gbk编码

这个gbk指cmd是用这个，当你用cmd进行数据的存入和读取的时候，自然会编码错误

要么就是把cmd改为utf8编码，但本人没尝试过

要不就是修改配置文件，在MySQL的安装目录下找到my.ini（这个文件是自己建立的设置文件，不是mysql自带的），把default-character-set的值改为gbk

这些设置只对MySQL提供的客户端（也就是cmd中用的那个）管用

设置完重启就好
